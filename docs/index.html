Overview

Smart Agriculture TinyML CNN Edge Computing Plant Disease Detection TensorFlow Lite OpenMV

Chroma-Sense is a novel TinyML model for classifying plant leaf diseases directly on microcontroller-class edge devices — cameras, drones, IoT sensors — without any cloud connection. Traditional pipelines send field imagery over the internet to cloud-hosted CNNs, introducing latency, security risks, and reliability problems in remote farmlands.

✖ Cloud CNN Pipeline

  • Images sent over internet to cloud servers
  • Latency from network round-trips
  • Security risk for proprietary crop data
  • Fails with poor/no rural connectivity
  • Heavy models: 100K–500K parameters

✓ Chroma-Sense on Edge

  • Inference runs locally on the sensor
  • Real-time results at 8.4 FPS
  • No data leaves the device
  • Works offline, anywhere in the field
  • Tiny model: just 15K parameters, 54 KB
💡

Key insight: Plant disease patterns — spots, rings, patches, mosaics — are semantically simple and consistent across RGB channels. Chroma-Sense processes R, G, and B channels sequentially through the same shared feature extractor, slashing RAM and parameter count while maintaining accuracy.

2

The Core Idea: Serial Multi-Channel Processing

How Plant Diseases Appear Per Channel

Each color channel of a leaf image carries simpler, sparser features than the combined RGB image. Consider Apple Cedar Rust — orange/yellow spots with a red ring and black center:

🍎
RGB Combined
Orange spot, red ring, black center — complex multi-color pattern
R
Red Channel
Bright yellow + red ring both visible (yellow = R+G)
G
Green Channel
Bright yellow region only; red ring absent
B
Blue Channel
Only the dark black center is visible

Each channel shows only simple shapes: a dot, a ring, a patch. And the same types of shapes appear across all three channels. A network that can detect a circular spot in grayscale works equally well on R, G, or B individually.

Because the same feature extractor is reused for all three channels, the parameter count drops dramatically. Because channels are processed one at a time, only one channel's activations ever live in RAM simultaneously — cutting peak memory by ~2/3 compared to standard parallel multi-channel approaches.

Why Global Max Pooling (not Average)?

Plant diseases appear randomly — a leaf may have 2 spots or 10. With average pooling, a leaf with only 2 sparse spots produces an activation of 0.2, while one with 10 dense spots produces 0.8. This variance causes misclassifications. After Int8 quantization, rounding amplifies the error further.

ℹ

Global Max Pooling focuses on the strongest activated spot regardless of disease count, producing a consistent signal of 1.0 for any diseased leaf — far more robust to quantization and disease instance variability.

Novel Contributions

💾

Memory Efficiency

Sequential processing reduces CNN width by ~2/3, cutting peak RAM by 25% vs. MobileNetV3.

🔢

Parameter Reduction

Shared extractor across all channels: just 15K parameters vs. 97K in the closest competitor.

🔎

Simplified Features

Grayscale-per-channel processing forces learning shapes/textures, not complex color correlations.

🔍

Explainability

Channel-specific feature maps embed spatial info — Grad-CAM clearly highlights disease regions.

Architecture

Feature Extractor (Shared, 7,352 params)

Processes a single grayscale channel (96×96×1). Uses SeparableConv2D (depthwise + pointwise) to minimize parameter count — the same technique used in MobileNet, adapted here for plant-disease semantics at a much smaller scale.

InputLayer (96, 96, 1) 0 ↓ Conv2D 3×3 (96, 96, 8) 80 params MaxPooling2D 2×2 (48, 48, 8) 0 ↓ SeparableConv2D (48, 48, 24) 228 params MaxPooling2D 2×2 (24, 24, 24) 0 ↓ SeparableConv2D (24, 24, 48) 1,416 params MaxPooling2D 2×2 (12, 12, 48) 0 ↓ SeparableConv2D (12, 12, 48) 2,746 params MaxPooling2D 2×2 (6, 6, 48) 0 ↓ SeparableConv2D (6, 6, 48) 2,746 params MaxPooling2D 2×2 (3, 3, 48) 0 ── Total: 7,352 parameters ──────────────────

Full Classification Model (~15K params)

The shared extractor runs on each channel separately, outputs are concatenated, and pointwise (1×1) convolutions fuse cross-channel information with a learned attention weighting before the classification head.

InputLayer (96, 96, 3) ↓ split Lambda R (96, 96, 1) ← Red Lambda G (96, 96, 1) ← Green Lambda B (96, 96, 1) ← Blue ↓ each → Feature Extractor (shared) (3, 3, 48) per channel ↓ concatenate Concatenate (3, 3, 144) ↓ Conv2D 1×1 (3, 3, 48) ← cross-channel fusion Conv2D 1×1 (3, 3, 16) ← compress ↓ GlobalMaxPooling2D(16,) Dense + Softmax (N classes,) ── ~15,164 total parameters ─────────────────
💡

Cross-channel attention via pointwise convolution: The 1×1 conv after concatenation performs a weighted linear sum of R, G, B features — e.g., 2R + 1G − 1B — restoring the color correlation lost during independent processing, while keeping parameters minimal.

Code Guide

1. Build the Shared Feature Extractor

CAI.ipynb — Feature Extractor
def build_feature_extractor(input_shape=(96, 96, 1)):
    inp = Input(shape=input_shape)
    x = Conv2D(8, (3,3), padding='same', activation='relu')(inp)
    x = MaxPooling2D((2,2))(x)
    x = SeparableConv2D(24, (3,3), padding='same', activation='relu')(x)
    x = MaxPooling2D((2,2))(x)
    x = SeparableConv2D(48, (3,3), padding='same', activation='relu')(x)
    x = MaxPooling2D((2,2))(x)
    x = SeparableConv2D(48, (3,3), padding='same', activation='relu')(x)
    x = MaxPooling2D((2,2))(x)
    x = SeparableConv2D(48, (3,3), padding='same', activation='relu')(x)
    x = MaxPooling2D((2,2))(x)
    return Model(inp, x)  # 7,352 params, reused 3×

2. Assemble the Full Model

CAI.ipynb — Full Chroma-Sense Model
def build_chroma_sense(num_classes):
    fe = build_feature_extractor()      # shared weights

    rgb = Input(shape=(96, 96, 3))
    r = Lambda(lambda x: x[:,:,:, 0:1])(rgb)
    g = Lambda(lambda x: x[:,:,:, 1:2])(rgb)
    b = Lambda(lambda x: x[:,:,:, 2:3])(rgb)

    # Same extractor, sequential inference
    rf = fe(r);  gf = fe(g);  bf = fe(b)

    x = Concatenate()([rf, gf, bf])        # (3,3,144)
    x = Conv2D(48, (1,1), activation='relu')(x)  # fuse
    x = Conv2D(16, (1,1), activation='relu')(x)  # compress
    x = GlobalMaxPooling2D()(x)
    out = Dense(num_classes, activation='softmax')(x)
    return Model(rgb, out)

3. Data Augmentation

CAI.ipynb — Augmentation
def augment_image(image, label):
    image = tf.image.random_flip_left_right(image)
    image = tf.image.random_flip_up_down(image)
    scale = tf.random.uniform([], 1.0, 1.2)
    new_size = tf.cast(96 * scale, tf.int32)
    image = tf.image.resize(image, [new_size, new_size])
    image = tf.image.random_crop(image, [96, 96, 3])
    image = tf.image.random_brightness(image, max_delta=0.2)
    return image, label

5. Edge Inference (OpenMV / MicroPython)

ei_image_classification.py — deployed as main.py on device
import sensor, image, time, tf, uos, gc

sensor.reset()
sensor.set_pixformat(sensor.RGB565)
sensor.set_framesize(sensor.QVGA)   # 320×240
sensor.set_windowing((240, 240))    # center crop
sensor.skip_frames(time=2000)       # auto-exposure settle

with open("/labels.txt") as f:
    labels = [l.rstrip('\n') for l in f]

net = tf.load("/trained",
    load_to_fb=uos.stat('/trained')[6] > (gc.mem_free() - (64*1024)))

clock = time.clock()
while True:
    clock.tick()
    img = sensor.snapshot()
    for obj in net.classify(img):
        preds = sorted(zip(labels, obj.output()), key=lambda x: -x[1])
        for label, conf in preds:
            print(f"{label}: {conf:.2f}")
    print(f"FPS: {clock.fps():.1f}")

Final results (Apple): Training accuracy 98.08%, validation accuracy 92.18% after fine-tuning. Validation loss 0.2587.

Two-Stage Training Strategy

  1. Phase 1 (250 epochs): Train all layers from scratch with Adam (lr=0.0005), save best checkpoint by validation accuracy.
  2. Phase 2 — Fine-tuning (10 epochs): Load best checkpoint, unfreeze final 65% of layers, retrain with reduced lr=0.000045 to sharpen feature representations without catastrophic forgetting.

Option C — Manual TFLite (OpenMV IDE)

  1. Copy trained.tflite → /trained on device
  2. Copy labels.txt → /labels.txt
  3. Copy ei_image_classification.py → main.py
  4. Connect in OpenMV IDE and run — classifications appear immediately at boot

Runtime inference (Python): For desktop testing, install pip install tensorflow numpy pillow and load trained.tflite via tf.lite.Interpreter. Remember to quantize input to Int8 using the model's scale/zero_point before running inference.
